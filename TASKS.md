# Task Breakdown

## Milestone 1: Define the project

- Finalize v1 scope
- Write `PRD.md`
- Write `ARCHITECTURE.md`
- Write `AI_WORKFLOW.md`
- Create the initial implementation plan

## Milestone 2: Choose and set up the stack

- Compare backend options briefly
- Record the backend choice in an ADR
- Scaffold the frontend project
- Scaffold the backend project
- Set up the database and local environment

## Milestone 3: Learn React with a local-only UI

- Create the initial app layout
- Build list and todo components with mock data
- Implement create, edit, complete, and delete interactions with local state
- Implement filtering with local state

## Milestone 4: Add persistence

- Create database tables for lists and todos
- Build API endpoints for lists
- Build API endpoints for todos
- Replace mock data with real API calls
- Handle loading and error states

## Milestone 5: Finish v1 behavior

- Support multiple lists end to end
- Support due dates
- Support plain-text notes
- Validate required fields
- Tighten empty-state behavior

## Milestone 6: Refactor intentionally

- Review which parts of the simple implementation feel awkward
- Decide whether additional libraries are justified
- Add ADRs for non-obvious decisions

## Milestone 7: Quality and documentation

- Add critical-path tests
- Improve UI polish
- Write or refine the README
- Refine architecture notes for interview readability

## Milestone 8: Ship and review

- Deploy the app
- Verify the deployed environment
- Write a retrospective on technical and workflow lessons

## Working Rule

Do not add features that are not in the PRD without an explicit scope decision.
